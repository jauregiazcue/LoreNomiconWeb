Texturing

In this project I am in charge of the lighting and texturing of this game, that means the mood and the overall looks of this game. My team and I had decided to go for a cartoon look with hard lines and flat colors therefore lighting is most important to bring out the most from what the artist will provide me. Since this project this engine uses tinyOBJ to insert models that means it does not provide information other than vertices, normal and UV. Due to this problem the artist will not be able to provide me with the TBN (tangent, bitangent and normal) calculation for the normal map for the texture and thus I take it upon myself to insert the calculation needed to calculate the TBN using the vertices, normal and UV in the Resource Manager (tangent) and the primitive shader (bitangent), and everything will be place together into a 3X3 matrix in the pixel shader. With this I can easily place the normal textures onto the object.



Lighting

For lighting, I mainly work in the primitive shader, pixel shader and the renderer to get everything showing in the mood and atmosphere the designer hoped for. In this game I worked on not only the directional light and spot light, I also worked on the fog effect. In this game since we are going for a dark horror vibe I decided to go for minimum directional lighting with a scaler on the intensity so that it does not look so bright in the game instead of ambient lighting.


The fog amount is first calculated based on the distance between the fragment and the camera position, considering fog start and range parameters. Fog is applied to the total output color based on the calculated fog amount using lerp.


For directional lighting I have fixed the direction of the lighting to one place and since there is no sunlight or moving light in this game there wasn’t a need to constantly change the direction of the light. Though in the future when more light is introduced into this game it can be easily changed as directional light is easily modifiable. In this game, diffuse lighting is calculated using Lambert’s cosine law, where the diffuse color is multiplied by the dot product between the light direction and the surface normal. As for the specular lighting, it is calculated using the Blinn-Phong reflection model. Both of these will output to show what you see on screen.


For Spot lighting I used the same concept as directional lighting but instead of fixed direction of the lighting it changes whenever the player turns and I added a calculation to calculate and limit light into a cone shape so it gives a flashlight effect.


Sound System

In the sound system it can simply split into 4 main sections, the audio port, the audio player, the audio library and the audio manager. Everything is placed together to make the audio work. This sound system contains the 3D sound system ( directional audio) and is able to be used in any game as it is part of the engine not the game.


Audio Manager is where the control manager is. This is where the system decides which audio file goes with which available player. This is also where everything audio related converged. It is also a place where all audio is recorded.


Audio port is a class for the type of audio. It is where the type of audio is created, initialized and tweaked.


Audio File is a class for reading and saving all the audio file pointers. It is an archive of all the data and its size that can be called upon when the audio needs to be played.


Audio Player is a class that can be created many times to play many different audios. It is a place where all the data will be temporarily stored until either the game ends or the end of the audio that is at which the audio data will be switched to a different state and ready for new audio and its data. There are 4 states at which the player can be in the Not Available state, the Initialised state, the Playing state and the Terminating state.


Audio Out 2 is a class that brings the audio port, audio player and audio file classes together. Audio Out 2 stores all the path to audio players, audio files and audio port so when an audio needs to be played all three classes can be called upon to make audio play.
